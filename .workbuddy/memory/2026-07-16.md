## 修复右上角绿色 Toast 一直显示/脉冲的 bug
用户反馈：右上角绿勾（成功提示）一直挂在屏幕上、渐隐渐现。

根因（ReservationManagement.jsx 第 946 行）：
- 渲染条件误写为 `showToast && (...)`。showToast 是 state 对象（始终 truthy），导致 Toast **永远渲染**；
- 外层 `animate-pulse` 持续透明度动画，造成"渐隐又出现"的观感。

修复：
- 显隐判断改为 `showToast.show`（真正的布尔开关）；
- 移除 `animate-pulse`，改为外层容器 `transition-opacity duration-300` + `showToast.show ? 'opacity-100' : 'opacity-0 pointer-events-none'`，实现操作成功时淡入、3 秒后淡出；
- 未操作时 Toast 完全不可见且不响应交互。

验证：npm run build 编译成功。

## 教训（通用）
Toast/弹层显隐务必用 state 里的布尔字段（如 `x.show`）做判断，绝不能直接用整个 state 对象（对象永远 truthy 会让它常驻）；需要"闪一下自动消失"的效果别用 `animate-pulse`（那是持续循环动画），用 `transition-opacity` + 布尔切换做淡入淡出。
