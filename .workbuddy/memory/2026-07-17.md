## 座位管理新增「按日期查看占用」功能
- **用户问题**：原来 `getSeatStatus` 用 `r.date >= today` 判断占用——某座位只要有未来任意一天的预约，今天就整日显示红色不可约（跨日期误判冲突）。用户希望选哪天就展示哪天的占用情况。
- **改动（src/components/SeatManagement.jsx）**：
  - 新增 `selectedDate` 状态（默认今天）。
  - `getSeatStatus / getSeatReservation / getSeatUserInfo` 增加 `date` 参数（默认 `selectedDate`），占用判断由 `r.date >= today` 改为 `r.date === date`（严格按所选日期）。
  - 顶部房间选择旁边新增「查看日期」date 选择器（min=今天）。
  - 信息卡片显示当前查看日期徽章（今天高亮蓝色）+ 说明文案。
  - 座位网格按 `selectedDate` 计算占用；occupied 卡片底部标签统一显示「已约」。
  - 点空闲座位弹预约框时，`formData.date` 默认带入 `selectedDate`（弹窗内仍可改）。
  - 座位详情弹窗按 `selectedDate` 查询；空闲提示改为"在 {日期} 暂无预约"+"切换日期可查看其他日期"。
- **效果**：切换日期即可查看/预约任意一天的座位占用，避免跨天误判冲突。固定座位（fixed）仍长期显示，不受日期影响。
- 构建通过。

## 路由级代码分割（减小首屏 bundle / 提升 LCP）
- **背景**：性能报告提示首屏打包了当前页面用不到的大量 JS（如座位页也加载了 ECharts），拉长 LCP（预估可省 ~1.8s）。
- **改动（src/App.js）**：
  - `import { useState, useEffect, useMemo, lazy, Suspense }`；把 6 个页面组件（Dashboard / SeatManagement / ReservationManagement / UserManagement / Settings / Minesweeper）由直接 import 改为 `React.lazy(() => import(...))`。
  - `renderContent` 调用外层用 `<Suspense fallback={<spinner/>}>` 包裹（仅包在已登录主壳的内容区，未登录的 Login/Register 仍走 eager 分支不进 Suspense）。
  - 外壳组件 Sidebar / Header / Login / Register 以及 RadioPlayer（含 RadioMiniBar 具名导出，被主壳每晚渲染）保持 eager，避免破坏电台迷你播放器。
  - **未**拆分 RadioPlayer 整页：因 RadioMiniBar 依赖同文件的 MiniBar/RadioPlayerModal，强拆有破坏风险且电台页无重型依赖，故保持常驻。
- **构建结果（代码分割生效）**：
  - 首屏 `main.js` 由约 490 kB 降至 **87.7 kB gzip**（瀑布标注 -402.29 kB）。
  - ECharts + 仪表盘拆为独立 chunk（204...chunk.js，约 373 kB），仅打开仪表盘时加载。
  - 其余页面各自独立小 chunk（0 / 336 / 597 / 970 / 457 / 453 等）。
- 结论：首屏只下载外壳+当前页 JS，其他页面按需加载，直接解决"当前页加载无用 JS"问题。
